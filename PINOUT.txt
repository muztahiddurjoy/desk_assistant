===============================================================================
 DESK ASSISTANT — PIN MAP
 Board : ESP32-S3 Dev Module (ESP32-S3-DevKitC-1)
 Last updated: 2026-07-12
===============================================================================

 Only one peripheral: the round display. Five signal pins, all inside GPIO 4..13.

 That block is chosen because on the ESP32-S3 it is entirely free of side jobs:

   - not strapping pins        (those are GPIO 0, 3, 45, 46)
   - not SPI flash             (GPIO 26..32)
   - not octal PSRAM           (GPIO 33..37 on R8 modules)
   - not native USB            (GPIO 19, 20)
   - not the onboard RGB LED   (GPIO 38 or 48, depending on revision)

 So nothing here fights the bootloader, and the same map works whether or not
 your module has PSRAM.


-------------------------------------------------------------------------------
 1. GC9A01 — 1.28" round IPS LCD, 240x240, 4-wire SPI
-------------------------------------------------------------------------------

 The 7-pin breakout. All seven pins are used; there is nothing else on the board
 to connect.

   DISPLAY PIN     ESP32-S3        NOTES
   -----------     ------------    ---------------------------------------------
   VCC             3V3             3.3 V only. 5 V will kill the panel.
   GND             GND
   SCL             GPIO 5          SPI clock
   SDA             GPIO 4          data in; panel is write-only, no MISO wire
   RES             GPIO 8          hardware reset
   DC              GPIO 7          data / command select
   CS              GPIO 6          chip select

   NO BACKLIGHT PIN. On this breakout the backlight is tied to VCC on the board:
   on whenever the panel is powered, at full brightness, not dimmable from
   firmware. TFT_BL is deliberately left undefined in platformio.ini — pins.h
   #errors the build if anyone puts it back. Any "brightness" on this panel has
   to be paid for in pixel values.

   SPI runs at 40 MHz on SPI2 (see USE_HSPI_PORT below).


-------------------------------------------------------------------------------
 2. Free pins
-------------------------------------------------------------------------------

   GPIO 9          free — never wired (would have been the backlight)
   GPIO 10         free — was WS2812 stick L
   GPIO 11         free — was WS2812 stick R
   GPIO 12         free
   GPIO 13         free

   The two WS2812 sticks have been removed from the project. Their pins, and all
   the 5 V / level-shifting / power-budget care they needed, are gone with them.

   Anything you add later (button, I2C, I2S mic) should stay inside 4..13.
   Everything outside that block is untouched by this project.


-------------------------------------------------------------------------------
 3. Two settings the S3 will not work without
-------------------------------------------------------------------------------

 Both are in platformio.ini. Both cost hours if you don't know them, because
 each one fails in a way that looks like broken hardware.

   USE_HSPI_PORT
     TFT_eSPI has no VSPI on the ESP32-S3 (SPI0/1 belong to the flash). Without
     an explicit port flag it binds to a bus that never starts, spiStartBus()
     returns NULL, and the first writecommand() inside tft.init() dereferences
     it: Guru Meditation / StoreProhibited / EXCVADDR 0x10, boot loop. Looks
     exactly like a dead panel. USE_HSPI_PORT selects SPI2.

   ARDUINO_USB_CDC_ON_BOOT=0
     This board talks over a CH343 USB-UART bridge (lsusb: 1a86:55d3) on
     /dev/ttyACM0. Setting this to 1 routes Serial to the S3's *native* USB
     peripheral instead — a port that isn't in use — so you get total silence on
     ttyACM0 while the firmware runs perfectly. Silent serial reads as a dead
     board. Only set it to 1 if you move the cable to the port marked "USB".


-------------------------------------------------------------------------------
 4. Display orientation
-------------------------------------------------------------------------------

 The panel is MIRRORED in hardware — MADCTL bit MX, set in applyMirror() in
 main.cpp — because it is viewed as a reflection (Pepper's-ghost / hologram rig).
 Everything drawn comes out left-right reversed on the glass, for free.

 Watch out: tft.setRotation() rewrites MADCTL from scratch and silently undoes
 this. Call applyMirror() after any setRotation(), or never call it again.


-------------------------------------------------------------------------------
 5. Keeping this file honest
-------------------------------------------------------------------------------

   The same numbers appear in three places:

     platformio.ini    build_flags   (TFT_eSPI reads its pins ONLY from here)
     include/pins.h    #defines      (everything the application code uses)
     PINOUT.txt        this file     (what you solder from)

   pins.h static_asserts itself against the build flags, so the first two cannot
   silently disagree — the build fails instead. This file is the one that has to
   be updated by hand. If you move a pin, move it here too.
